# 数据库管理脚本

本目录包含 Pink Ledger 项目的数据库管理脚本。

## 📁 脚本说明

### init-database.js - 数据库初始化脚本

**⚠️ 重要：此脚本仅用于项目首次初始化！**

该脚本会：

1. ✅ 检查数据库文件是否已存在
2. 🚫 如果已存在，停止运行（避免误操作）
3. 🔧 如果不存在，创建全新数据库并初始化：
   - 创建所有数据表
   - 初始化系统分类数据
   - 显示数据库状态

**核心特性：**
- 🔒 **安全保护**：自动检测已存在的数据库，避免误操作
- 🎯 **专注首次初始化**：不处理数据库更新和迁移
- 📊 **简单直接**：无需考虑增量更新和数据保留

## 🚀 使用方法

如果你使用 OrbStack / Docker Compose 启动服务，容器入口脚本会自动执行 `node scripts/init-database.js`，通常不需要再手动初始化。此时数据库默认位于 `./data/database.sqlite`。

### 首次初始化数据库

```bash
# 在项目根目录下执行
node scripts/init-database.js
```

**执行结果：**
- 如果数据库**不存在**：创建全新数据库并初始化数据
- 如果数据库**已存在**：提示已初始化，停止运行

### 查看帮助信息

```bash
node scripts/init-database.js --help
```

## 📋 使用场景

### 1. 项目首次部署

```bash
# 1. 安装依赖
pnpm install

# 2. 配置环境变量
cp .env.example .env

# 3. 初始化数据库（仅首次需要）
node scripts/init-database.js

# 4. 启动服务器
npm start
```

### 2. 开发环境重置数据库

如果需要完全重建数据库（删除所有数据）：

```bash
# 1. 手动删除数据库文件
rm database.sqlite

# 2. 重新初始化
node scripts/init-database.js
```

### 3. 多次运行脚本

如果不小心多次运行脚本：

```bash
$ node scripts/init-database.js

╔═══════════════════════════════════════════════════════╗
║  ⚠️  数据库已存在！                                   ║
╚═══════════════════════════════════════════════════════╝

检测到数据库文件已存在，无需再次初始化。

💡 提示：
   - 如果要启动服务器，请直接运行: npm start 或 pnpm start
   - 如果要重建数据库，请先手动删除数据库文件
```

脚本会自动检测并停止运行，不会影响现有数据。

## ⚠️ 注意事项

1. **仅用于首次初始化**：此脚本专门用于项目首次部署，不适用于数据库结构更新
2. **自动保护机制**：如果检测到数据库已存在，脚本会自动停止运行
3. **权限要求**：确保脚本对数据库文件所在目录有写入权限
4. **环境变量**：确保 `.env` 文件配置正确，特别是 `DB_PATH` 配置
5. **独立运行**：此脚本独立于服务器启动，必须在服务器运行前执行
6. **数据库更新**：如需更新数据库结构（添加字段等），请手动编写迁移脚本或使用专业的数据库迁移工具

## 🔍 故障排查

### 问题：提示"数据库已存在"但实际需要重建

**解决方法**：
```bash
# 先删除旧数据库文件
rm database.sqlite
# 或者根据提示的路径删除
rm /path/to/your/database.sqlite

# 然后重新运行初始化脚本
node scripts/init-database.js
```

### 问题：数据库连接失败

**解决方法**：
- 检查 `.env` 文件中的 `DB_PATH` 配置
- 确保数据库文件所在目录存在且有写入权限
- 确保没有其他进程占用数据库文件

### 问题：表创建失败

**解决方法**：
- 检查 Node.js 版本是否 >= 14.0.0
- 确保所有依赖已正确安装：`pnpm install`
- 查看详细错误信息，检查模型定义是否正确

## 📝 环境变量

脚本会读取以下环境变量：

```env
# 数据库文件路径（可选，默认为项目根目录下的 database.sqlite）
DB_PATH=./database.sqlite

# Node 环境（可选，默认为 development）
NODE_ENV=development
```
